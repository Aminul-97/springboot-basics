Q: I want the 'create-customer-dialog' to be pre-rendered, dialogs open
  instantly, and there’s no need to send additional requests. How can I do
  that? Just share me the detailed steps. Don't make any change to the code.
  
  
• Use this exact approach for pre-rendered create-customer-dialog (no code changes made by me).

  1) Controller: prepare create form model in /customer

  File: CustomerCtrl.kt (/home/aminul/Greg/nexbal/src/main/kotlin/no/reai/nexbal/ctrl/CustomerCtrl.kt)

  @GetMapping
  fun customers(model: Model): String {
      val customerItems = customerSvc.listCustomers().map { it.toCustomerListItemResponse() }
      model.addAttribute("customersResponse", CustomersResponse(items = customerItems))
      model.addAttribute("createCustomerRequest", CreateCustomerRequest())
      return "customers"
  }

  2) Template: remove hx-get from "new customer" button, only open dialog

  File: customers.html (/home/aminul/Greg/nexbal/src/main/resources/templates/customers.html)

  <wa-button
      size="small"
      class="custom-btn"
      onclick="document.querySelector('#customer-new-dialog').open = true;"
      pill>
      <wa-icon slot="start" name="user-plus"></wa-icon>
      <th:block th:text="#{customer.list.newLink}"></th:block>
  </wa-button>

  3) Template: pre-render fragment inside dialog body

  File: customers.html (/home/aminul/Greg/nexbal/src/main/resources/templates/customers.html)

  <wa-dialog id="customer-new-dialog">
      <div id="customer-new-dialog-body"
           th:replace="~{fragments/create-customer-dialog :: content}">
      </div>
  </wa-dialog>

  4) Optional: full-page validation error flow (recommended for pre-rendered mode)

  File: CustomerCtrl.kt (/home/aminul/Greg/nexbal/src/main/kotlin/no/reai/nexbal/ctrl/CustomerCtrl.kt)

  @PostMapping
  fun createCustomer(
      @Valid @ModelAttribute("createCustomerRequest") createCustomerRequest: CreateCustomerRequest,
      bindingResult: BindingResult,
      model: Model,
  ): String {
      if (bindingResult.hasErrors()) {
          val customerItems = customerSvc.listCustomers().map { it.toCustomerListItemResponse() }
          model.addAttribute("customersResponse", CustomersResponse(items = customerItems))
          model.addAttribute("openCreateCustomerDialog", true)
          return "customers"
      }

      customerSvc.createCustomer(createCustomerRequest.toCreateCustomerPayload())
      return "redirect:/customer"
  }

  Then in customers.html, open dialog on load when needed:

  <wa-dialog id="customer-new-dialog" th:attr="open=${openCreateCustomerDialog}">
      <div id="customer-new-dialog-body"
           th:replace="~{fragments/create-customer-dialog :: content}">
      </div>
  </wa-dialog>
