
# ✅ 1️⃣ Basic HTMX Button → Call Spring Boot Function

### 📌 Controller (Kotlin)

```kotlin
@Controller
class UserController {

    @PostMapping("/do-action")
    fun doAction(model: Model): String {
        println("Function executed!")

        model.addAttribute("message", "Action completed successfully")
        return "fragments :: result"
    }
}
```

---

### 📌 Thymeleaf Template (HTML)

```html
<button 
    hx-post="/do-action"
    hx-target="#result"
    hx-swap="innerHTML">
    Click Me
</button>

<div id="result"></div>
```

---

### 📌 Fragment (Thymeleaf)

```html
<div th:fragment="result">
    <p th:text="${message}"></p>
</div>
```

---

### 🔥 What happens?

1. Button is clicked.
2. HTMX sends POST request to `/do-action`.
3. Spring Boot executes `doAction()`.
4. Returned fragment replaces `#result` div.

No JavaScript needed.

---

# ✅ 2️⃣ If You Want to Execute a Pure Backend Function (No UI Update)

If you only want to trigger a backend function without swapping HTML:

```html
<button 
    hx-post="/do-action"
    hx-swap="none">
    Execute
</button>
```

Spring still runs the function — just no DOM update.

---

# ✅ 3️⃣ If You Actually Want to Run a JavaScript Function

HTMX is server-first, but you *can* combine it with JS.

```html
<button onclick="myFunction()">Click</button>

<script>
function myFunction() {
    console.log("JS function executed")
}
</script>
```

Or use HTMX event hooks:

```html
<button 
    hx-post="/do-action"
    hx-on::after-request="myFunction()">
    Click
</button>
```

---

# ✅ 4️⃣ More Advanced: Passing Data

```html
<button 
    hx-post="/do-action"
    hx-vals='{"userId": "123"}'
    hx-target="#result">
    Click
</button>
```

Controller:

```kotlin
@PostMapping("/do-action")
fun doAction(@RequestParam userId: String): String {
    println("User ID: $userId")
    return "fragments :: result"
}
```

---

# 🎯 Recommended Pattern (Best Practice)

Since you're building server-driven UI:

✔ Keep logic in Spring Boot
✔ Use HTMX to trigger endpoints
✔ Return partial fragments
✔ Avoid unnecessary JS

This keeps your app clean and maintainable.
